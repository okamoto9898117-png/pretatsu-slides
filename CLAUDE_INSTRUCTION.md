## 🎯 プレタツが目指すスライドの定義

> **「現代日本のハイセンスなビジネス媒体や広告ポスター、おしゃれな日本の雑誌のように、知的でクリーン、でも感性に刺さる編集デザイン」**

### 目指すべき3つの状態
1. **視聴者が直感的に理解できる** — 見た瞬間にわかる
2. **飽きずに最後まで見たくなる** — 次のスライドが気になる
3. **最後に【ゴール】へ動きたくなる** — CTAで行動を起こしたくなる

## 🚫 絶対にやってはいけないこと（禁止ルール）

| 禁止事項 | 理由 |
|---------|------|
| 長文・説明文をそのまま載せる | 読む気が失せる。見出し・短文・図解に変換する |
| 同じレイアウトを連続で使う | 単調になる。毎スライドで構図を変える |
| 「左に文字、右にキャラ」の固定構図 | AI感が出る。多様なレイアウトを使う |
| 最下部に不自然な帯・フッター | 未使用領域に見える。四辺まで背景を伸ばす |
| ロゴ・注釈・ページ番号・極小文字 | 情報ノイズ。削除する |
| 画像内に文字を描き込む | テキストはテキスト要素として配置する |
| 下端だけ白く抜ける | スライド全面を使い切る |
| 見出しだけ追っても話が通じない構成 | 見出し→本文の流れを一貫させる |

## ✅ 必ず守るべきデザインルール（プレタツ標準）

### レイアウト
- **1スライド1メッセージ** — 1枚で伝えることは1つだけ
- **表紙を必ず作る** — 第一印象がすべて
- **必要枚数に自然に分割** — 詰め込まず、分割を恐れない
- **スライド全面を使い切る** — 四辺まで背景・構成要素を伸ばす
- **余白はデザインとして使う** — 空白=高級感
- **毎スライドでレイアウト構図を変える** — 単調さを排除

### テキスト
- **長文禁止** — 見出し・短い本文・図解・余白で伝える
- **見出しだけ追っても話が通る** — 見出しが「目次」になっている状態
- **体言止め** — 「〜する」ではなく「〜の実現」
- **テキストはテキスト要素として配置** — 画像に文字を焼き込まない

### ビジュアル
- **フルブリード写真 or グラデーション** — 背景を四辺まで伸ばす
- **ブランドガイドに準拠** — 会社・サービスのカラーを優先
- **高品質なUnsplash写真** — 解像度1920px以上を使用
- **構図のバリエーション** — 全画面・2分割・3分割・中央集中を交互に使う

## 📐 設計原則（全テーマ共通・必ず守ること）

### ① 1スライド1メッセージ
- 1枚で伝えることは**1つだけ**
- タイトルだけ読めば内容がわかる状態にする

### ② KISS法則（Keep it Short and Simple）
- **体言止め**を使う（「〜する」→「〜の実現」）
- 箇条書きは3〜5項目まで
- 不要な装飾・重複テキストは削除

### ③ 配色の黄金比率
- ベースカラー：**70%**
- メインカラー：**25%**
- アクセントカラー：**5%**
- 色数は**最大3色**まで

### ④ 数字の見せ方（世界最高水準）
- 重要な数字は**32〜180pt**で巨大に表示
- 数字の横に単位（%・円・社）を添える
- 背景はダーク、数字はホワイト or ゴールド

### ⑤ 写真の使い方（Editorialスタイル）
- 写真はフルブリード（スライド全面）で使用
- 写真の上に黒透明オーバーレイを重ねてテキストを読みやすく
- `sizing: { type: "cover", w: 10, h: 5.625 }` を必ず使用

### ⑥ 余白の贅沢な使い方
- 最小マージン：0.5"以上
- 詰め込まない・空白を恐れない
- 余白こそが「高級感」を生む

## ⚡ Claudeへの実行指示

フォームの内容を読み取り、以下を実行すること：

1. **テーマを確認** → カラーパレットを決定
2. **Editorialテーマの場合** → Unsplash写真をhttpsでダウンロードして使用
3. **スクリプトを作成** → `/tmp/pptx_build/` でPptxGenJS Node.jsスクリプトを生成
4. **実行** → `node スクリプト名.js`
5. **保存先** → `/Users/[ユーザー名]/My Campany/presentation_[会社名].pptx`
6. **GitHubにアップロード** → `okamoto9898117-png/presentation-blueprint` リポジトリ

**使用ライブラリ：**
- `pptxgenjs` → PPTXファイル生成
- `react-icons/fa` → アイコン（FAアイコン）
- `sharp` → SVGをPNGに変換
- `https` → 写真ダウンロード（Node.js標準）

**注意事項：**
- 色コードに`#`を付けない（PptxGenJS仕様）
- shadowオブジェクトは使い回し禁止（毎回`makeShadow()`で生成）
- テキストの`margin: 0`を使って位置を正確に合わせる
